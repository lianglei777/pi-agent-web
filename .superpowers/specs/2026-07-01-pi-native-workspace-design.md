# Po Agent Web：Pi 原生工作台视觉重构设计

日期：2026-07-01
状态：已完成方向确认，等待书面规范复核

## 1. 背景与目标

Po Agent Web 已具备稳定的开发者工作台结构，但当前冷灰绿色主题、偏圆润的控件和缺少视觉锚点的聊天空状态没有形成明确、耐看的品牌语言。此次重构参考 [Pi Coding Agent 官网](https://pi.dev/) 的视觉系统，同时保留 Po Agent Web 的产品结构和工作效率。

最终方向为“Pi 原生 Po 工作台”：

- 视觉层尽量接近 Pi 的纸张网格、暖灰面板、深蓝灰画布、低饱和蓝色、细边框、低圆角、衬线标题与等宽标签。
- 产品层保留 Po Agent Web 的项目、会话、聊天、文件、模型、Skills 和审查工作流。
- 不复制官网导航、营销文案、安装命令、虚假终端交互或纯展示型动效。
- 浅色和深色主题作为同一设计系统同时重做。
- 建立可长期约束后续功能的设计规范，避免视觉方向继续漂移。

## 2. 设计原则

1. **工作台优先**：视觉借鉴不能降低信息密度、操作效率或状态可读性。
2. **Pi 语言，Po 身份**：使用相近的设计语法，但采用独立的 Po 像素标识和产品结构。
3. **阅读区安静**：网格只作为画布语境，不出现在长文本、代码、表单、文件树和高密度列表背后。
4. **边界建立层级**：主要依靠背景层次与细边框，普通面板不使用阴影。
5. **状态高于品牌色**：错误、警告、成功和危险操作继续使用独立语义色，并辅以文字、图标或边界。
6. **真实交互**：不增加看似可运行但没有真实行为的命令面板、按钮或能力入口。
7. **系统化而非局部换肤**：颜色、字体、圆角、边框、阴影和状态必须通过语义 token 与共享组件落地。

## 3. 范围

### 3.1 本次包含

- 浅色与深色主题 token。
- 全局画布、网格、字体、圆角、边框、阴影和动效规则。
- 独立的像素 `Po` 标识及其在侧栏、欢迎页和 favicon 中的使用。
- 共享 UI 组件的视觉状态。
- 工作台侧栏、顶部栏、聊天主区、输入框、消息和执行过程。
- 模型配置、Skills、文件工作区、列表、表单和 Dialog。
- 中英文、三个目标桌面宽度及无障碍验收。
- `DESIGN.md`、设计 sidecar 和视觉契约测试同步。

### 3.2 本次不包含

- 业务流程、API 合同、服务端能力或数据结构变更。
- 官网式 Home、Docs、News、Packages 或 Models 导航。
- 虚假的终端窗口、命令 Tab、`RUN` 按钮或安装命令。
- 营销 Hero、能力卡片墙、轮播、粒子、光效或背景漂移动画。
- 新增字体库、动画库、组件库或其他前端依赖。
- 移动端布局；继续支持项目规定的最小宽度 `1024px`。

## 4. 主题与色彩

主题以 Pi 官网当前色彩关系为依据，映射到项目已有的语义 token。具体实现允许为 WCAG AA 和长时间阅读做小幅数值校准，但不得改变整体色相与层级关系。

### 4.1 核心色板

| 语义角色 | 浅色 | 深色 | 用途 |
|---|---|---|---|
| 深层背景 | `#dacbc2` | `#0d1116` | 画布最深层、少量外围层次 |
| 主画布 | `#ebe7e4` | `#161d27` | 应用主背景与网格底色 |
| 主面板 | `#f4f2f0` | `#212730` | 侧栏、顶部栏、阅读面板 |
| 次级面板 | `#eef1f3` | `#252f3d` | 输入、列表分组、代码和工具区域 |
| 主文字 | `#252f3d` | `#ebe7e4` | 标题、正文和主要控件文字 |
| 次级文字 | `#5c5752` | `#9fa4ab` | 描述、元数据和次要状态 |
| 强调色 | `#4b607c` | `#6a9fcc` | 焦点、选中、主要动作和实时状态 |

边框、hover、selected、disabled、focus ring 和消息背景由上述核心色板派生为独立语义 token。功能组件不得直接使用核心色值或自行混色。

### 4.2 状态色

- `destructive`：删除、凭据移除、断开连接和不可逆操作。
- `error`：失败与无效输入。
- `warning`：副作用、费用和风险提示。
- `success`：保存、安装和验证成功。

状态色在两个主题中分别调校。任何状态都不能只依赖颜色表达。

### 4.3 网格

- 网格仅应用于聊天主画布的空白外围和其他明确的画布型背景。
- 使用细网格、主网格和交叉点三层低对比图案，整体接近 Pi 官网，但强度以长时间使用舒适为上限。
- 聊天阅读列、配置页、文件树、表单、代码块和浮层必须使用不透明纯色表面覆盖网格。
- 深色主题的网格对比度低于浅色主题，避免产生视觉闪烁。
- 网格是唯一允许的多层线性渐变用途，不引入装饰性渐变。

## 5. 字体系统

复用项目已有字体，不增加依赖：

- **展示字体**：Playfair Display / Noto Serif SC。只用于欢迎标题、少量一级页面标题和主要 Dialog 标题。
- **界面字体**：Inter、系统无衬线与现有中日韩回退字体。用于正文、导航、表单、按钮、列表和聊天内容。
- **等宽字体**：Noto Sans Mono 与现有等宽回退字体。用于路径、模型、状态、代码、元数据和短标签。

约束：

- 衬线字体不进入长聊天正文、按钮、表单、文件树或会话列表。
- 等宽标签允许适度字距，但中文不得强制大写或使用影响识别的宽字距。
- 正文默认保持紧凑、稳定的行高；长文本阅读列控制在约 65–75 个拉丁字符的视觉宽度。

## 6. 形状、边界、阴影与动效

- 普通面板：`0px`；需要表达独立边界的小型容器：`2px`。
- 按钮、输入框和文本区域：`4px`。
- Dialog、Dropdown、Select content、Tooltip 和 Popover：`6px`。
- 徽标、状态点和开关轨道可以使用胶囊或圆形。
- 普通面板、消息、列表和 Composer 不使用阴影。
- 只有真实浮层使用低透明度的小范围阴影。
- Hover 只改变背景、文字或边框，不改变几何位置和尺寸。
- 动效仅用于状态过渡、展开折叠和必要的进度反馈，使用现有 `150–220ms` 时长。
- 禁止卡片上浮、内容逐字展示、网格漂移和装饰性页面入场动画。
- `prefers-reduced-motion: reduce` 继续将非必要动画降为近即时。

## 7. 品牌标识

创建独立的几何像素 `Po` 标识：

- 以单色几何块构成，浅色和深色背景均可识别。
- 不直接描摹 Pi Logo 的字形或块状结构。
- 在小尺寸 favicon 和侧栏中仍有清晰轮廓。
- 主要使用位置为侧栏顶部、聊天欢迎页和 favicon。
- 不将标识作为背景纹理或在同一视图中重复装饰。

标识应以项目原生 SVG 或代码可维护的矢量形式实现，不引入图片处理依赖。

## 8. 页面结构

### 8.1 应用外壳

保留现有三栏结构：

```text
项目与会话侧栏 | 聊天或配置主区 | 可选文件工作区
```

保留面板折叠、宽度调整、聊天挂载状态和配置导航行为。不引入官网顶部导航。

### 8.2 左侧栏

- 使用纯色主面板，不显示网格。
- 顶部使用小尺寸 Po 标识和现有“新对话”入口。
- 项目与会话继续使用高密度列表。
- 选中态组合低饱和蓝灰背景、字重和边界，不依赖单一色块。
- 主题和语言控制继续固定在底部。
- 保留宽度调整、折叠、键盘访问和禁用原因提示。

### 8.3 顶部栏

- 高度保持现有 `40px`，维持桌面工具栏密度。
- 显示当前会话或配置页标题。
- 项目路径、分支和文件区开关使用紧凑文字或图标。
- 使用主面板底色与单条细边框建立层级。
- 不加入大 Logo、官网导航或装饰文案。

### 8.4 聊天空状态

空状态只包含：

1. 独立像素 Po 标识。
2. 一句简短的衬线标题。
3. 一行产品说明。
4. 真实可用的聊天输入框。

不加入命令 Tab、`RUN` 按钮、能力卡片墙或自动播放文案。现有工作区中的快捷提示卡片删除改动保持不被恢复，除非未来有独立的产品需求和真实使用证据。

### 8.5 对话状态

- 网格只保留在消息阅读列外围。
- 消息阅读列使用纯色背景，避免文字覆盖网格。
- 用户消息使用轻微蓝灰表面，不使用气泡尾巴。
- 助手最终回答保持开放式正文，减少重复容器。
- Thinking、工具调用和执行过程继续使用现有可折叠语义结构。
- 代码块允许比正文阅读列更宽，但不能破坏整体布局。

### 8.6 输入框

- 视觉上接近开发者控制台的秩序感，但保持真实文本编辑器行为。
- 使用明确边框、`4px` 圆角和无阴影表面。
- 输入正文使用界面字体；模型、Thinking、附件和状态使用等宽小字。
- 发送按钮继续使用真实图标与既有行为，不伪装成终端命令。
- 保留运行中、排队、停止、错误、图片能力和禁用原因逻辑。

### 8.7 配置页与文件工作区

- 模型、Skills 和文件区采用 Pi 文档页式的线性结构。
- 使用页面标题、工具栏、分区面板、列表和详情层级，避免每段信息都成为 Card。
- 表单、代码预览、文件树和长文本区域使用纯色背景。
- 保留现有危险操作确认、未保存变更保护、副作用提示和忙碌状态。

## 9. 共享组件规范

### 9.1 Button

- Primary：蓝灰实底，只用于当前区域的主要提交动作。
- Secondary / Outline：透明或次级表面配细边框。
- Ghost：用于工具栏和列表操作。
- Destructive：保持独立危险色与具体动作文案。
- 每种变体覆盖 default、hover、focus-visible、active、selected、disabled 和 loading。

### 9.2 Input、Textarea 与 Select

- 使用纯色次级面板、明确边框和可见焦点环。
- 错误状态同时提供边框、文字和可访问描述。
- Placeholder 对比度可读，但明显低于输入值。
- 禁用状态必须阻止交互，并在需要时提供具体原因。

### 9.3 Card、列表与面板

- Card 只用于确实独立且可单独理解的内容单元。
- 默认结构优先使用列表、分区和面板。
- 普通容器无阴影；选中态不改变布局。

### 9.4 Dialog、Dropdown 与 Tooltip

- 使用完整细边框、`6px` 圆角和轻微浮层阴影。
- Dialog 保留禁止背景点击和 Escape 意外关闭的规则。
- Dropdown 与 Select 维持紧凑行高，选中状态结合背景和标记。
- Tooltip 只承载短说明；复杂信息使用可聚焦的正式内容区域。

### 9.5 聊天消息与代码

- 消息类型通过结构、标签和表面区分，不依赖头像或角色化装饰。
- 代码、路径、模型名和状态使用等宽字体。
- 代码块使用 Pi 式深浅终端面板，但不添加窗口控制圆点或虚假终端装饰。
- 成功、警告和错误同时使用文字、图标或边界。

## 10. 无障碍与国际化

- 目标为 WCAG 2.1 AA。
- 保留键盘导航、语义标签、可见焦点和屏幕阅读器文本。
- 主题切换同时更新原生 `color-scheme`。
- 所有新增或修改的用户可见文案同步更新中英文词典。
- 中文标题不依赖英文大小写效果；中文等宽标签单独校准字距。
- 验证浅色、深色、中英文在 `1024px`、`1440px` 和 `1920px` 下的布局。
- 滚动条可以降低视觉存在感，但必须保持可发现、可拖动和足够命中区域。

## 11. 实施顺序

1. **视觉基础**：更新双主题 token、字体、网格、圆角、边框、阴影和规范文档。
2. **共享组件**：统一现有 UI primitives 的全部交互状态。
3. **工作台框架**：重做侧栏、顶部栏、面板边界、调整手柄和文件区外观。
4. **聊天体验**：重做欢迎页、Composer、消息、执行过程、代码块和错误状态。
5. **配置页面**：统一模型、Skills、列表、表单、详情页和确认 Dialog。
6. **全局验收**：处理跨页面不一致、主题遗漏、对比度、国际化和目标宽度问题。

每一步只调整视觉和必要的组件结构，不改变业务行为。发现需要业务变更时必须单独提出，不并入本次视觉重构。

## 12. 测试与验收

### 12.1 自动检查

- 更新视觉基础契约测试，锁定核心浅色与深色 token。
- 更新共享组件视觉契约，覆盖允许的圆角、阴影和状态类。
- 保留现有功能、交互、状态和 i18n 测试。
- 扩展现有视觉契约测试，检查代表性功能组件没有新增一次性颜色、装饰渐变和大阴影；不为此引入新的自定义检查工具。
- 运行 `npm run check`。
- 由于涉及 Next.js 全局样式、字体、布局和生产渲染，运行 `npm run build`。

### 12.2 浏览器验收矩阵

关键页面包括：聊天空状态、进行中对话、包含工具调用的完成态、模型配置、Skills、文件工作区和危险确认 Dialog。

每个关键页面验证：

- 浅色 / 深色。
- 中文 / 英文。
- `1024px` / `1440px` / `1920px`。
- 键盘焦点与 reduced motion。
- 正常、hover、selected、disabled、loading 和 error 状态。

### 12.3 完成标准

- 第一眼能够识别 Pi 的暖纸张、网格、蓝灰、衬线和等宽视觉语言。
- 产品仍明确是 Po Agent Web 工作台，而非 Pi 官网复制品。
- 浅色和深色共享同一设计语言，并分别调校。
- 网格不影响聊天正文、表单、文件树和代码阅读。
- 不新增依赖、虚假功能、营销区块或装饰动效。
- 不改变聊天、会话、模型、Skills、文件和 Dialog 的业务行为。
- 功能组件不新增一次性颜色、阴影或圆角。
- 目标语言与宽度无截断、重叠和明显布局空洞。
- 自动检查、生产构建和浏览器验收全部通过。
- 最终关键页面截图经统一评审后，核心 token 与禁止项进入稳定维护状态。

## 13. 对现有分析文档的取舍

`docs/po-agent-web-pi-style-analysis.md` 作为设计输入保留，其中关于纸张网格、像素标识、衬线与等宽字体、低饱和蓝灰和细边框的判断被采纳。以下建议不进入最终方案：

- 玻璃模糊和半透明 Composer。
- 命令 Tab、`RUN` 按钮与能力卡片墙。
- 卡片 hover 位移与逐字输入动画。
- 仅定义浅色主题的 token 替换。
- 用 CSS 阴影临时拼接品牌 Logo。

本规范是后续实施和长期维护的视觉来源；分析文档不作为实现合同。
